# خطوات النشر السريع (Quick Deployment)

يمكنك نشر هذا المشروع مجانًا وبسهولة باستخدام خدمة **Render**، وهي تدعم تشغيل تطبيقات Docker مباشرة.

## الخطوات:

### 1. رفع الكود إلى GitHub
1. أنشئ حسابًا على [GitHub](https://github.com) (إذا لم يكن لديك).
2. أنشئ "New Repository" وسمّه `ICDLAdminPortal`.
3. ارفع ملفات المجلد `ICDLAdminPortal` إلى هذا المستودع (يمكنك استخدام GitHub Desktop أو سطر الأوامر).

### 2. النشر على Render
1. سجل الدخول إلى [Render](https://render.com).
2. اضغط على **New +** واختر **Web Service**.
3. اختر **Build and deploy from a Git repository** واربط حساب GitHub الخاص بك.
4. اختر مستودع `ICDLAdminPortal` الذي أنشأته.
5. في الإعدادات:
   - **Name**: اختر اسمًا لموقعك (مثلاً `icdl-portal`).
   - **Region**: اختر الأقرب إليك (مثلاً Frankfurt).
   - **Runtime**: اختر **Docker**.
   - **Free Plan**: تأكد من اختيار الخطة المجانية.
6. **Environment Variables** (متغيرات البيئة):
   اضغط على "Add Environment Variable" وأضف التالي:
   - `ICDL_ADMIN_SECRET`: ضع كلمة سر طويلة ومعقدة (لتأمين التوكن).
   - `ICDL_USERS_FILE`: `/var/tmp/users.json` (للتخزين المؤقت) أو اتركها فارغة إذا كنت لا تهتم بضياع البيانات عند إعادة التشغيل.

7. اضغط **Create Web Service**.

### 3. التجربة
- انتظر بضع دقائق حتى ينتهي الـ Build.
- سيظهر لك رابط في الأعلى (مثلاً `https://icdl-portal.onrender.com`).
- افتح الرابط وستعمل لوحة التحكم!

### ملاحظة هامة حول البيانات
في الاستضافة المجانية (مثل Render)، ملفات النظام "مؤقتة" (Ephemeral). هذا يعني أن ملف `users.json` سيُحذف إذا أعاد الخادم تشغيل نفسه (وهذا يحدث تلقائيًا).
لحل هذه المشكلة بشكل دائم، يجب ربط التطبيق بقاعدة بيانات خارجية، ولكن للتجربة السريعة، الطريقة الحالية كافية.
